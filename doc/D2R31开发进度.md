## 2026-06-13：Huffman 编码表不兼容 — 待提取

### 问题

D2R 3.1 存档的物品 code 使用了与 D2R 2.x **不同的 Huffman 编码表**。当前硬编码的表（`src/core/D2Item.cpp:42-58`，37个字符）编码 `"hp1 "` 得到的位序列在 3.1 存档中完全找不到。v2.5 存档解析正常，v3.1 存档在第 7 个物品开始解码偏移导致 D2Error(6)。

### Huffman 表位置

Huffman 表硬编码在 `src/core/D2Item.cpp:42-58`，定义了 37 个可编码字符（`0-9 a-z 空格`）的 Huffman 树结构。该表对所有 D2R 2.x 版本有效，但对 3.1 无效。

### 提取方案

#### 方案 A：从游戏二进制提取（1 个文件即可）

**需要的文件**（任一即可）：

```
<D2R 3.1 安装目录>/
├── D2R.exe          ← 主程序，优先
├── Game.exe         
├── D2R.dll          
└── *.dll            ← 任意游戏 DLL
```

**提取方法**：在二进制文件中搜索类似当前 HUFFMAN[] 数组的树结构字节序列：
- 当前表的特征：以 `1,1,1,1,1` 开头，0x00 分隔符，ASCII 字符穿插
- 用 Python 脚本扫描文件，找到匹配模式的区域
- 验证：用候选表编码已知 item code 并在存档中匹配

#### 方案 B：从存档反向推导（需要多个单物品存档）

**需要的文件**（约 8-10 个 .d2s 文件）：

| 文件名 | 物品 | item code | 覆盖的字符 |
|--------|------|-----------|-----------|
| `hp1.d2s` | 生命药水 (Health Potion) | `hp1` | h, p, 1 |
| `mp1.d2s` | 法力药水 (Mana Potion) | `mp1` | m, p, 1 |
| `rv1.d2s` | 活力药水 (Rejuvenation Potion) | `rv1` | r, v, 1 |
| `tsc.d2s` | 辨识之书 (Tome of Identify) | `tsc` | t, s, c |
| `isc.d2s` | 辨识卷轴 (Scroll of Identify) | `isc` | i, s, c |
| `key.d2s` | 钥匙 (Key) | `key` | k, e, y |
| `gld.d2s` | 金币 (Gold) | `gld` | g, l, d |
| `yps.d2s` | 解毒药剂 (Antidote) | `yps` | y, p, s |

每个存档的要求：
- D2R 3.1 新建 1 级角色
- 背包中**只放 1 个目标物品**（其余全部清空）
- 不要装备任何物品

**推导方法**：
1. 已知 Huffman 编码的输入（item code）和输出（存档中的位序列）
2. 不同物品 code 共享字符时，该字符的编码必须一致
3. 通过交集求解：hp1 和 mp1 共享 `p1`，差集得 `h` 和 `m` 的编码
4. 收集足够覆盖 37 个字符的存档即可重建完整表

### 推荐优先级

1. ⭐ **方案 A** — 如果有 D2R 3.1 游戏安装目录，一个文件即可
2. ⭐⭐ **方案 B** — 如果没有游戏文件，在游戏中创建单物品存档最可靠
