# 暗黑2重制版3.2版本支持改动方案

## 一、项目现状分析

### 1.1 当前支持的版本
项目目前支持以下版本：
- **经典版本**: 1.00-1.06 (0x47), 1.07/Expansion 1.08 (0x57), Standard 1.08 (0x59), 1.09 (0x5C), 1.1x (0x60)
- **重制版**: Resurrected (0x61), PTR2.4 (0x62), PTR2.5 (0x63), 术士君临/Reign of the Warlock (0x64)

### 1.2 版本标识机制
项目使用存档文件中的 `dwVersion` 字段（DWORD类型）来识别版本：
- 版本号采用十六进制表示
- 每个大版本更新递增版本号
- 版本检测函数位于 `D2Version.h`

### 1.3 版本相关的核心文件
1. **D2Version.h** - 版本检测函数和有效版本列表
2. **DlgCharBasicInfo.cpp** - 版本显示名称映射
3. **D2S_Struct.h/cpp** - 存档结构定义和读写逻辑
4. **Diablo Edit2.h/cpp** - 职业和技能数据定义
5. **language.txt** - 多语言文本资源（需编译为language.dat）

## 二、3.2版本支持改动方案

### 2.1 确定版本号
根据现有版本号规律：
- 0x61: Resurrected 基础版
- 0x62: PTR 2.4
- 0x63: PTR 2.5
- 0x64: 术士君临 (Reign of the Warlock)

**建议3.2版本使用版本号: 0x65**

### 2.2 代码改动清单

#### 2.2.1 D2Version.h
**文件路径**: `Diablo Edit2/D2Version.h`

**改动内容**:
```cpp
// 在现有版本检测函数后添加
inline BOOL IsPtr32AndAbove(DWORD dwVersion) { return dwVersion >= 0x65; }

// 在 IsValidVersion 函数的 switch 语句中添加
inline BOOL IsValidVersion(DWORD dwVersion) {
    switch (dwVersion)
    {
        case 0x65:  // PTR 3.2 或正式版 3.2
        case 0x64:  // 术士君临 (Reign of the Warlock)
        case 0x63:
        // ... 其他版本
    }
}
```

**改动原因**: 
- 添加版本检测函数，便于后续代码中判断是否为3.2及以上版本
- 将0x65添加到有效版本列表，使编辑器能够识别和加载3.2版本的存档文件

#### 2.2.2 DlgCharBasicInfo.cpp
**文件路径**: `Diablo Edit2/DlgCharBasicInfo.cpp`

**改动内容**:
在 `UpdateUI` 函数的 switch 语句中添加（约第150行）：
```cpp
void CDlgCharBasicInfo::UpdateUI(const CD2S_Struct& character)
{
    switch (character.dwVersion) {
        case 0x65:m_sVersion = _T("3.2"); break;  // 或 "PTR3.2"
        case 0x64:m_sVersion = _T("术士君临"); break;
        // ... 其他版本
    }
}
```

**改动原因**: 在界面上正确显示存档的版本信息

#### 2.2.3 README.md
**文件路径**: `README.md`

**改动内容**:
```markdown
Tested in 1.09, 1.10, 1.13, Diablo II: Resurrected (PTR 2.4/2.5/2.6/3.2).
```

**改动原因**: 更新文档说明支持的版本列表

### 2.3 可能需要的额外改动

#### 2.3.1 存档结构变化检测
**检查点**: `D2S_Struct.h` 和 `D2S_Struct.cpp`

**操作步骤**:
1. 获取3.2版本的实际存档文件（.d2s文件）
2. 对比2.6版本存档，检查是否有结构变化：
   - 字段偏移是否改变
   - 是否新增字段
   - 数据格式是否调整

**常见变化点**:
- 新职业添加（如之前的术士）
- 新技能添加
- 新物品属性
- 新任务数据
- 雇佣兵数据结构

#### 2.3.2 物品数据更新
**检查点**: `Generate Data/itemdata.txt` 和相关代码

**操作步骤**:
1. 检查3.2版本是否新增物品类型
2. 检查现有物品属性是否有变化
3. 如有变化，更新 `itemdata.txt` 并重新生成 `itemdata.dat`

#### 2.3.3 属性系统更新
**检查点**: `Generate Data/property.txt` 和 `MetaData.cpp`

**操作步骤**:
1. 检查3.2版本是否新增物品属性
2. 检查现有属性的参数格式是否变化
3. 如有变化，更新 `property.txt` 并重新生成 `property.dat`
4. 可能需要在 `MetaData.cpp` 中添加版本特定的属性处理逻辑

#### 2.3.4 技能系统更新
**检查点**: `Generate Data/language.txt` 和 `Diablo Edit2.cpp`

**操作步骤**:
1. 检查3.2版本是否为现有职业新增技能
2. 检查是否有新职业（类似术士的情况）
3. 如有新技能，更新 `language.txt` 中的技能名称
4. 如有新职业，需要：
   - 更新 `CLASS_NAME_SIZE` 常量
   - 添加新职业的 `CLASS_SKILL_INDEX` 数组
   - 更新职业名称和技能树名称

#### 2.3.5 版本特定逻辑处理
**检查点**: 所有使用 `IsPtr24AndAbove`、`IsPtr25AndAbove`、`IsRotWAndAbove` 的代码

**操作步骤**:
1. 搜索所有版本检测函数的使用位置
2. 评估3.2版本是否需要特殊处理
3. 如需要，添加 `IsPtr32AndAbove` 的判断逻辑

**已知使用位置**:
- `D2Item.cpp`: 物品数据读写（第153, 166, 346, 402行等）
- `D2S_Struct.cpp`: 存档结构读写（第340行）
- `DlgCharItems.cpp`: 界面显示逻辑（第534行等）

## 三、实施步骤

### 3.1 最小改动方案（仅支持读取）
如果3.2版本的存档结构与2.6版本完全相同，只需：

1. **修改 D2Version.h**
   - 添加 `case 0x65` 到 `IsValidVersion` 函数

2. **修改 DlgCharBasicInfo.cpp**
   - 添加版本显示名称映射

3. **更新 README.md**
   - 添加3.2到支持版本列表

4. **测试**
   - 使用3.2版本存档文件测试读取和编辑功能

### 3.2 完整支持方案（包含新特性）
如果3.2版本有结构变化或新特性：

1. **获取3.2版本信息**
   - 获取官方3.2版本更新说明
   - 获取实际的3.2版本存档文件样本
   - 分析存档文件的二进制结构

2. **分析差异**
   - 对比2.6和3.2版本存档的十六进制数据
   - 识别新增字段、修改字段
   - 记录偏移量变化

3. **更新数据文件**
   - 更新 `itemdata.txt`（如有新物品）
   - 更新 `property.txt`（如有新属性）
   - 更新 `language.txt`（如有新文本）
   - 运行 `Generate Data` 工具重新生成 .dat 文件

4. **修改代码**
   - 按照2.2节的改动清单修改代码
   - 添加版本特定的读写逻辑（如需要）
   - 更新界面显示逻辑（如需要）

5. **编译和测试**
   - 编译项目
   - 使用多个3.2版本存档测试
   - 验证读取、编辑、保存功能
   - 验证修改后的存档在游戏中正常工作

### 3.3 数据文件重新生成流程

如果需要更新数据文件：

1. **编辑源文件**
   - 修改 `Generate Data/itemdata.txt`
   - 修改 `Generate Data/property.txt`
   - 修改 `Generate Data/language.txt`

2. **编译生成工具**
   - 打开 `暗黑II.sln` 解决方案
   - 编译 `Generate Data` 项目

3. **运行生成工具**
   - 运行 `Generate Data.exe`
   - 自动生成 `Diablo Edit2/itemdata.dat`
   - 自动生成 `Diablo Edit2/property.dat`
   - 自动生成 `Diablo Edit2/language.dat`

4. **重新编译主程序**
   - 编译 `Diablo Edit2` 项目
   - 测试新生成的数据文件

## 四、风险评估

### 4.1 低风险场景
- 3.2版本仅是2.6版本的bug修复版本
- 存档结构完全兼容
- 只需添加版本号识别

**改动量**: 3个文件，约10行代码

### 4.2 中风险场景
- 3.2版本新增少量物品或属性
- 存档结构基本兼容，仅扩展部分字段
- 需要更新数据文件

**改动量**: 5-10个文件，约50-100行代码

### 4.3 高风险场景
- 3.2版本新增职业或大幅改动技能系统
- 存档结构有重大变化
- 需要重构部分读写逻辑

**改动量**: 10+个文件，约200+行代码

## 七、后续维护建议

### 7.1 版本管理策略
建议为每个支持的游戏版本创建对应的编辑器版本标签：
- v0.30-d2r-3.2
- v0.29-d2r-2.6
- 便于用户选择合适的编辑器版本

### 7.2 自动化测试
建议建立自动化测试框架：
- 准备各版本的测试存档文件
- 自动验证读取和写入功能
- 回归测试确保不破坏旧版本支持

### 7.3 文档维护
- 及时更新README中的支持版本列表
- 记录每个版本的特殊处理逻辑
- 维护版本差异对照表

## 八、总结

支持暗黑2重制版3.2版本的核心工作是：

1. **确定版本号**: 建议使用 0x65
2. **最小改动**: 修改3个文件（D2Version.h, DlgCharBasicInfo.cpp, README.md）
3. **关键验证**: 获取实际3.2版本存档，验证结构兼容性
4. **扩展改动**: 根据实际差异更新数据文件和处理逻辑
5. **充分测试**: 确保读取、编辑、保存功能完整可用

建议先实施最小改动方案进行初步测试，根据测试结果决定是否需要更深入的改动。
